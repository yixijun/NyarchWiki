    安装应用程序

    在 Nyarch Linux 上，您可以通过多种方式安装应用程序，这些方式称为"打包格式"。 本页面解释了不同打包格式之间的差异以及如何安装和管理它们。

    支持的软件包：

    1. Arch Linux 软件包 (pacman, .tar.zst)
    2. Flatpaks (商店, .flatpakref, .flatpak)
    3. AppImages (.appimage)
    4. Arch 用户仓库 (yay, nyay)

    Arch Linux 软件包

    （几乎）每个 Linux 发行版都有一个包管理器：它帮助您在计算机上安装、删除和更新软件包。

    在 Arch Linux 上，包管理器是 pacman。

    由于多种原因，pacman 应该只通过终端并使用管理员权限使用。

    以下是 pacman 的一些基本用法，如果您想要更详细的说明，请查看 Arch Wiki 上的 pacman

    如何安装软件包

    sudo pacman -S package

    例如，要安装 firefox，您可以输入

    sudo pacman -S firefox

    如何删除软件包

    sudo pacman -R package

    如果您还想删除其依赖项，可以使用 -Rns。

    如何更新所有软件包

    sudo pacman -Syu

    如何搜索软件包

    sudo pacman -Ss package

    Flatpaks

    Flatpak 是一种特殊的打包格式，允许您在沙盒环境中安装应用程序。

    这意味着您可以轻松管理应用程序的权限，就像在智能手机上一样。

    Flatpaks 有一些优缺点： - 优点： - 它们可以在每个 Linux 发行版上运行 - 您可以控制它们的权限 - 通过商店轻松安装 - 您可以在没有管理员权限的情况下安装应用程序 - 缺点： - 与原生软件包相比，它们占用更多磁盘空间 - 对于一些不适合在沙盒中运行的软件，它们有时会出现问题 - 终端程序的使用通常需要执行一些额外的步骤

    我应该什么时候安装 flatpak？

    • 当软件主要应该作为 Flatpak 运行时（例如 Bottles）
    • 当您安装不信任的软件时（您可以控制其权限）
    • 当它是 GTK/Gnome 应用程序时（因为 Nyarch 附带了它们的运行时，安装速度很快）
    • 当软件不在终端中运行时

    安装 Flatpak

    Gnome Software

    您可以使用 Gnome Software 来查找、安装、卸载和更新 flatpak 应用程序。

    从 Flathub

    您可以在 Flathub 上找到 flatpaks。从那里安装应用程序会让您的浏览器下载一个 .flatpakref 文件，点击它会在 Gnome Software 中打开应用程序

    注意：flathub 上的每个应用程序也都在 Gnome Software 中

    命令行

    flatpak install application.id

    例如，通过 Flatpak 安装 firefox

    flatpak install org.mozilla.firefox

    使用命令行，您还可以安装 .flatpak 文件。

    要做到这一点，您只需运行

    flatpak install file.flatpak

    此外，双击 .flatpak 文件会打开 Gnome Software 来安装它。

    管理 Flatpak 权限

    您可以使用 Flatseal 管理 flatpak 权限。

    Appimages

    Appimages 是一种可以在每个 Linux 发行版上运行的特殊格式。 - 优点： - 它们可以在每个 Linux 发行版上运行 - 缺点： - 它们不与其他应用程序共享库 - 它们通常较慢 - 通常它们不支持主题

    如果有其他打包格式可用，通常不建议使用 appimages。 仅在没有替代方案时使用 appimages。

    安装和管理 Appimages

    下载 appimage 后，您可以双击它，它将启动 GearLever。

    GearLever 是一个简单的软件，可帮助您安装和卸载 appimages。 您可以选择运行一次 appimage 而不安装，或将其添加到应用程序菜单。

    从命令行运行 appimages

    您也可以直接从命令行运行 appimage： 1. 给它执行权限

    chmod +x application.appimage
    1. 运行它
    ./application.appimage

    Arch 用户仓库

    AUR（Arch 用户仓库）是一个由社区驱动的大型软件包商店，用于 Arch Linux 官方不支持的软件包。如果一个应用程序存在，您很可能会在 AUR 中找到它。

    Nyarch Linux 完全兼容 AUR，因为它使用默认的 Arch Linux 仓库。

    警告：AUR 上的软件包由用户上传，因此它们可能不安全。建议始终从软件包构建中检查它们是否可信。

    安装软件包

    要安装 AUR 软件包，您需要一个 AUR Helper。

    默认情况下，Nyarch Linux 附带 yay 作为默认的 AUR helper。（或者 nyay，因为 lol nya）

    要安装 AUR 软件包，您可以运行

    yay -S package

    例如，

    yay -S zen-browser-bin

    更新软件包

    yay

    您可能会遇到同一程序的多个软件包。这是 Vesktop 的一个例子：

    基于后缀：

    • -git：此软件包是从 Git 仓库中的最新代码构建的。它可能不稳定，但您将获得最新的功能和更新。此外，它将编译软件包

    • -bin：此软件包是二进制软件包，这意味着它是预编译的并且可以直接使用。您不需要从源代码构建它

    • 如果没有后缀，通常意味着该软件包是"发布"版本。换句话说，它是软件的最新稳定版本，没有任何后缀或前缀。
